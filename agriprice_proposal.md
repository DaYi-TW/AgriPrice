# AgriPrice Proposal

> 中文名稱：農價通  
> 專案類型：iOS 農產品行情查詢與供應商成交查詢 App  
> 目標使用者：親朋好友、農產品供應商、需要快速查看批發市場行情的人  
> 版本：MVP / POC Proposal  
> 更新日期：2026-05-13

---

## 1. 專案背景

AgriPrice 是一個針對台灣農產品批發市場行情設計的 iOS App，主要資料來源為 AMIS 農產品批發市場交易行情站。

本專案的核心目標不是打造大型公開服務，而是提供給少量親朋好友使用，因此系統設計以「低成本、易維護、夠穩定」為優先，不導入昂貴的雲端資料庫。

---

## 2. 專案目標

AgriPrice 要解決兩個主要需求：

1. 快速查詢特定農產品在所有市場的行情
2. 讓供應商查詢今日實際成交金額

使用者可以透過 AMIS 官方品項代號，例如 `FV4 辣椒 朝天椒`，查詢指定日期區間內各市場價格、交易量與趨勢。

---

## 3. 核心價值

### 3.1 一次看所有市場

傳統查詢流程通常需要逐一選市場或查表。AgriPrice 的設計重點是：

```text
輸入品項代號 + 日期區間
→ 一次顯示所有市場價格
→ 依均價 / 交易量 / 漲跌排序
```

### 3.2 今日成交查詢

供應商可以輸入：

```text
供應商代號
小代號
密碼
```

查詢今日實際成交結果，包含：

- 今日成交總額
- 總重量
- 平均單價
- 各市場成交金額
- 各件成交明細

### 3.3 不依賴 GCP 資料庫

因為使用人數少，本專案採用低成本架構：

```text
iOS App
↓
GCP Cloud Run API
↓
AMIS
```

行情紀錄、收藏品項、最近查詢與趨勢資料存在 iPhone 本機，不使用 Cloud SQL / Firestore。

## 4. 目標使用者

### 4.1 主要使用者

- 農產品供應商
- 農場家人
- 需要快速查看批發行情的親朋好友

### 4.2 使用情境

| 情境 | 說明 |
|---|---|
| 今日行情查詢 | 查看某品項今天在所有市場的價格 |
| 價格比較 | 比較台北、板橋、台中、高雄等市場行情 |
| 今日成交查詢 | 查詢自己今天賣出多少錢 |
| 趨勢觀察 | 查看近 7 / 30 / 90 日價格變化 |
| 常用品項快速查詢 | 常查品項排在前面，一鍵查詢 |

## 5. MVP 功能範圍

### 5.1 首頁

首頁以資訊卡片呈現：

- 今日最關注行情
- 全市場均價
- 今日漲跌
- 最高市場
- 最低市場
- 今日成交摘要
- 主要功能入口
- 常用品項快捷

### 5.2 市場行情查詢

使用者可選擇：

- AMIS 品項代號
- 日期區間，預設為今天～今天

查詢結果顯示：

- 全市場平均價
- 區間最高價
- 區間最低價
- 市場列表
- 各市場上價 / 中價 / 下價
- 各市場交易量
- 漲跌幅

### 5.3 品項選擇

品項選擇採用 Bottom Sheet 形式：

- 點擊綠色品項卡片開啟品項選單
- 以 AMIS 官方品項代號為主
- 常用品項可加星號
- 星號品項自動排序在前面

範例：

```text
FV4 辣椒 朝天椒
FV1 辣椒 紅小
FV2 辣椒 青小
LA1 甘藍 初秋
SG5 大蒜 蒜頭
```

### 5.4 日期區間選擇

日期查詢採用區間設計：

```text
開始日期 ～ 結束日期
```

預設：

```text
今天 ～ 今天
```

快捷選項：

- 今天
- 本月
- 近 7 日
- 近 30 日
- 近 90 日

### 5.5 趨勢分析

趨勢頁包含：

- 價格折線圖
- 交易量長條圖
- 市場比較
- 區間最高 / 平均 / 最低
- 簡易趨勢摘要

### 5.6 今日成交查詢

使用者輸入：

- 供應商代號
- 小代號
- 密碼

查詢後顯示：

- 今日成交總額
- 總重量
- 平均單價
- 各市場成交總額

點擊市場後可查看：

- 各件品項
- 重量
- 單價
- 成交金額

## 7. 系統架構

```text
SwiftUI iOS App
  ├─ SwiftData：行情歷史、收藏品項、最近查詢
  ├─ Keychain：供應商登入資訊，可選
  ├─ Swift Charts：趨勢圖
  ↓
GCP Cloud Run FastAPI
  ├─ AMIS Proxy
  ├─ HTML Parser / API Adapter
  ├─ Response Normalizer
  ↓
AMIS 農產品行情網站
```

---

## 8. 為什麼不使用 GCP 資料庫

本專案使用人數少，主要給親友使用，因此不建議第一版使用 Cloud SQL 或 Firestore。

原因：

1. 降低 GCP 成本
2. 減少維運負擔
3. 避免資料庫權限與備份管理
4. 查詢資料可存在本機 SwiftData
5. 未來使用者變多再升級後端資料庫

---

## 9. 安全設計

### 9.1 密碼不存後端

供應商密碼不存到 GCP。

若 iOS 需要記住密碼，使用：

```text
iOS Keychain
```

禁止使用：

```text
UserDefaults 儲存密碼
```

### 9.2 Cloud Run 不保留敏感資料

Cloud Run 只負責代理查詢與格式轉換，不儲存供應商帳號密碼。

## 13. 結論

AgriPrice 的 MVP 不需要做成大型雲端系統。最適合目前需求的做法是：

```text
SwiftUI iOS App
+ SwiftData 本機資料
+ Cloud Run FastAPI Proxy
+ AMIS 查詢
```

這樣可以兼顧低成本、可用性與未來擴充性。
